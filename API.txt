1. GET /items (เส้นทางดึงข้อมูล)

เส้นนี้เปรียบเสมือน "ตาสับปะรด" ของระบบครับ

    หน้าที่: วิ่งไปที่ตาราง items ใน MySQL แล้วหยิบทุกอย่างออกมาส่งให้หน้าเว็บ

    การทำงาน: เมื่อ Frontend เรียกเส้นนี้ Backend จะใช้คำสั่ง SELECT * FROM items

    ผลลัพธ์: ได้ข้อมูลแบบ JSON Array (รายการของทั้งหมด) เพื่อเอาไปสร้างเป็นตารางในหน้าเว็บ

    เหมาะสำหรับ: ใช้โชว์รายการของในหน้าแรก หรือใช้เช็คว่าของชิ้นไหนว่าง (Available) บ้าง

	1.1 get /items/ID
	หน้าที่ ดึงข้อมูลอุปกรณ์รายชิ้น โดยระบุ ID

2. POST /borrow (เส้นทางบันทึกการยืม)

เส้นนี้คือ "หัวใจ" ของระบบ และเป็นเส้นที่ซับซ้อนที่สุด (Transaction)

    หน้าที่: ทำ 2 อย่างพร้อมกันแบบ "แพ็กคู่"

        Insert: เพิ่มบันทึกลงในตาราง borrowing_logs (ใคร ยืมอะไร เมื่อไหร่)

        Update: เปลี่ยนสถานะในตาราง items จาก Available → Borrowed

    เงื่อนไข: ถ้าขั้นตอนใดขั้นตอนหนึ่งพัง (เช่น ไฟดับ หรือพิมพ์ไอดีผิด) ระบบจะ Rollback คือยกเลิกทั้งหมดเพื่อไม่ให้ข้อมูลค้างคา

    ข้อมูลที่ต้องส่ง: employee_id, item_id, admin_id, note

3. POST /return (เส้นทางบันทึกการคืน)

เส้นนี้คือ "จุดปิดจบ" ของกระบวนการยืม

    หน้าที่: ทำ 2 อย่างพร้อมกันเช่นกัน

        Update Log: ไปหาแถวใน borrowing_logs แล้วเติมวันที่คืน (return_date) ลงไป

        Update Item: เปลี่ยนสถานะในตาราง items จาก Borrowed → Available เพื่อให้คนต่อไปมายืมได้

    ข้อมูลที่ต้องส่ง: log_id (เพื่อจะได้รู้ว่าคืนของรอบไหน) และ item_id (เพื่อเปลี่ยนสถานะของ)

4. API get/history ดูประวัติการยืม-คืนทั้งหมด 
	หน้าที่ โชว์ประวัติการยืม

5.API get/employees
	หน้าที่ โชว์ข้อมูลในตาราง employees สามารถกำเนินข้อมูลที่จะดึกมาได้
6.API get /employees/:id
	ดึงข้อมูลพนักงานรายบุคคล โดยระบุ ID
